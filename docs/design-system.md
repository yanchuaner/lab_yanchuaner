# 燕中科创实验室设计系统

本文是 `lab.yanchuaner.cn` 的 UI 实施标准。它约束视觉语言、组件边界、主题、响应式、交互和无障碍；新增页面必须遵循本文，而不是从现有页面复制一份再局部改色。

## 1. 设计目标

燕中 Lab 是学习与共建工具，不是营销站。界面应当：

- 清晰：用户在首屏就能判断“学什么、怎么开始、在哪里参与项目”。
- 克制：动效用于表达状态和方向，不与正文争夺注意力。
- 可持续：亮暗主题、中英文和移动端必须由同一组件自然适配。
- 有温度：技术语言准确，但不制造门槛；鼓励完成第一件小作品。

视觉关键词为“开放实验台、工程蓝图、可靠信号”。亮色接近纸张与工作台，暗色接近低照度实验室；两者共享层级关系，不是两套独立皮肤。

## 2. 令牌与颜色

颜色唯一来源是 `src/app/globals.css` 的 `:root` 与 `.dark`。业务组件只能使用语义变量：

| 令牌 | 用途 |
| --- | --- |
| `--bg` | 页面背景 |
| `--surface` | 导航、卡片、输入框等一级表面 |
| `--surface-muted` | 次级表面、代码区、弱强调背景 |
| `--surface-strong` | 进度槽、分隔性填充 |
| `--text` | 主文字 |
| `--text-muted` | 说明、元数据和辅助标签 |
| `--line` | 默认边框与分隔线 |
| `--brand` | 主操作、当前状态、关键链接 |
| `--accent` | 技术信息与辅助图形 |
| `--success` / `--info` | 状态语义，不能仅靠颜色传意 |

规则：

- 禁止在 `src/components` 和页面文件中新增十六进制、RGB、HSL 裸色值。
- 禁止用 `white`、`black` 或固定灰阶模拟主题。
- 透明色使用 `color-mix()` 与语义令牌组合。
- 动态宽度、坐标等非颜色值可以使用内联 `style`；颜色不可以。
- 文本与背景至少满足 WCAG AA。正文不低于 `4.5:1`，大字号不低于 `3:1`。

## 3. 字体、密度与层级

- 正文字体使用系统无衬线字体栈；代码与机器状态使用等宽字体。
- 字号不随视口连续缩放。移动与桌面在明确断点切换。
- 页面只允许一个 `h1`；卡片标题从 `h2` 或 `h3` 顺序进入，不跳级。
- 正文建议每行 45–75 个中文字符或 55–85 个拉丁字符。
- 内容页正文行高为 `1.7–1.85`，工具栏和元数据可更紧凑。
- 卡片圆角上限 `8px`；按钮与输入框采用 `7px`，避免过度胶囊化。

## 4. 布局

全站采用三种页面骨架：

1. 探索首页：叙事入口 + 分类地图 + 推荐内容 + 共建项目。
2. 目录页：桌面左侧分类导航 + 主内容；移动端隐藏侧栏并保留搜索。
3. 详情页：主阅读区 + 辅助信息栏；窄屏改为单列，关键操作提前。

断点约定：

| 范围 | 行为 |
| --- | --- |
| `> 900px` | 完整导航、多列目录、详情双栏 |
| `621–900px` | 手机菜单、两列卡片、保留部分双栏 |
| `<= 620px` | 单列卡片、纵向主操作、详情单列 |

所有页面必须满足：`documentElement.scrollWidth === documentElement.clientWidth`。固定格式的进度条、图标按钮和卡片插图应有稳定尺寸，避免状态变化引起布局跳动。

## 5. 组件边界

基础组件集中在 `src/components/ui.tsx`，当前公共组件如下：

- `Button`：表单命令。
- `ButtonLink`：导航型主次操作。
- `Badge`：难度和状态，不承担按钮行为。
- `SectionHeading`：首页内容区标题、说明和可选动作。

页面壳与业务组合组件保持分离：

- `SiteShell` 只管理全局导航、页脚与主题/语言入口。
- `CatalogPage` 管统一目录结构与筛选。
- `GuidePage` 和 `ProjectPage` 分别表达教程与项目，不交叉复用业务语义。
- 内容数据位于 `src/lib/content.ts`，界面组件不得自行定义项目或教程文案。

新增抽象的门槛是：至少有两个真实调用点，且抽取后能统一状态、可访问性或主题行为。仅为了缩短 JSX 不创建组件。

## 6. 交互与状态

- 按钮使用动词；导航入口使用名词。
- hover 只能增强已有线索，关键信息不能只在 hover 出现。
- 搜索、筛选和分页要保留明确空状态。
- 外链需要可识别的外链图标；未接入的链接不得伪装成可用功能。
- 进度必须同时显示文字或数值，不能只显示色条。
- 加载完成前保持布局尺寸稳定；异步错误必须给出原因与恢复入口。
- 焦点样式不得移除。键盘用户应能依次访问导航、主操作和正文链接。

## 7. 动效

- 动效优先使用 `transform` 与 `opacity`，避免高频修改布局属性。
- 首页实验台旋转只承担环境氛围，不承载必须读取的信息。
- 页面切换不使用长时间遮罩；单次过渡建议 `120–240ms`。
- 必须尊重 `prefers-reduced-motion`，关闭循环动画并压缩过渡时间。
- Canvas 或持续动画未来接入时，必须同时处理 `visibilitychange`、视口可见性与卸载清理。

## 8. 双语与主题

- 界面固定文案只进入 `src/lib/i18n.ts`；内容标题与摘要使用 `{ zh, en }`。
- 组件通过 `t(key)` 和 `item.title[locale]` 读取，不写 `locale === "zh"` 文案分支。
- 中文缺失或英文缺失都视为内容未完成，不以机器翻译在运行时兜底。
- `html.lang` 必须随语言同步；主题与语言偏好保存于本地。
- 防闪烁脚本在 React 水合前应用暗色 class，根节点保留 `suppressHydrationWarning`。
- 英文导航按最长标签测试；空间不足时切换手机菜单，不压缩到重叠。

## 9. 无障碍与验收

每个页面至少检查：

- 语义标题顺序、唯一 `h1`、可用 landmark。
- 图标按钮有随语言变化的 `aria-label`。
- 纯装饰图形设置 `aria-hidden`，信息图形有文字等价物。
- 触控目标不小于 `36px`，主要移动操作建议不小于 `42px`。
- 375px、768px、1440px 下无溢出、遮挡或不可达操作。
- 中文/英文 × 亮色/暗色四种组合均通过视觉检查。

提交前运行：

```bash
pnpm typecheck
pnpm lint
pnpm audit:ui
pnpm build
```

## 10. 演进原则

在第三个燕中前端项目出现相同组件需求前，不建立跨仓库 UI 包。当前先保持令牌命名和组件契约一致；确认真实复用面后，再抽取 `@yanchuan/ui`，避免过早绑定各子站的信息密度与产品定位。
